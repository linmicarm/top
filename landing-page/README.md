# 🖥️ odin landing page

a fully hand-coded landing page built as part of **[the odin project](https://www.theodinproject.com/lessons/foundations-landing-page)** foundations curriculum. the goal was to recreate a provided design from scratch using only html and css, with a focus on flexbox layout.

---

## 🧠 concepts practiced

- flexbox layout (containers, alignment, spacing)
- page sectioning with semantic html
- css custom properties and color theming
- typography with google fonts
- linking external stylesheets
- recreating a design from a visual reference

---

## 🛠️ how it's made

**tech used:** html, css

this project is a static landing page built section by section from a provided design mockup. no frameworks, no shortcuts — just html structure and css flexbox doing the heavy lifting. each section was tackled individually to keep things manageable and intentional.

## ⚡ optimizations
 
if i were to revisit this project, i'd:
 
- make the layout **fully responsive** using media queries and mobile-first thinking
- refactor repetitive css into cleaner, reusable custom properties
- improve **accessibility** with better semantic elements and aria labels where needed
- optimize images for faster load times

---

## 💭 lessons learned

- building from a visual reference teaches you to slow down and *really* look at spacing and proportion
- flexbox clicked more once i stopped guessing and started reasoning about which element was the container
- committing section by section made the whole thing feel less overwhelming
